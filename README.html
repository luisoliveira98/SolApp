SolApp

Arquitetura

No desenvolvimento da aplicação foi usada Android Architectural Components que consiste em fazer a separação da interface de utilizador com a obtenção de dados a mostrar na UI, de forma a criar uma maior abstração na aplicação.

A arquitetura consiste na criação de uma View Model que faz pedidos à base de dados, esta à escuta de alterações na base de dados de forma a atualizar a UI com informação mais recente.

Base da dados

Na criação da base de dados foi usado a entididade WeatherByLocal, onde é guardado as informações sobre o estado do tempo por dia. Depois foram criadas as classes de Database e a interface Dao, onde sao especificadas as operações de acesso à base de dados.

WeatherRepository

Nesta classe é onde é feito o pedido à API externa (IPMA API), onde o pedido é feito se os dados guardados tiverem mais de 15min.

Para fazer o pedido à API é utilizado a biblioteca Volley

Observações:

A aplicação não coorre como esperado uma vez que que não esta a guardar os dados na base de dados, está toda ela preparada para o fazer, mas devido a algum problema que não consegui detetar não guarda os dados na bd, fazendo com que a aplicação não estaja como esperado.
